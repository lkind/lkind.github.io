探寻GAE

真的好久没写东西了. 有很多原因, 生活节奏一直比较紧, 不停地在电气行业探寻和吸取知识. 再者, octopress的安装还是太折腾了, 其中要涉及mac下面的ruby版本, llvm-gcc, 和各种奇怪的依赖. OK, 等你真的折腾成功了, 然后就是奇奇怪怪的中文编码问题, 自己还要修改代码. 然后好不容易稳定一段时间了, github pages从com迁移到io, 就是嫌你不够折腾. 然后我就尝试了hexo, 这个东西我真的是看着作者一点点写出来的, 作者是个台湾高中生, 前端牛人, 当然了, 现在是大学生了. 知道作者是因为他写了一个很漂亮的octopress主题. hexo这个东西是nodejs的, nodejs的依赖安装方式还是能接受的, 但是貌似我一直部署失败就没鸟过了, 后来就丢到heroku上面了, 但是我不是很喜欢heroku, 而且heroku访问太慢了.

反正不管怎么说, 现在好歹能写东西了, 尝试过ghost, 很不错, 但是需要自己额外购买一台VPS, 成本略高. 我现在坐等hexo更新, disqus被莫名替换成facebook comment了.

回归正题.

Google appengine

我相信这货唯一的吸引力跟heroku一样, 免费. 所以如果成功了的话, 诱惑还是很大的.

我这里讲的是python, 其他语言的就绕道把. 今天上半年想把python代码部署到网上以做展示用, 那个时候图快, 用的Django, 东西也没写几天. 然后想起来Google Appengine貌似免费, 具体机制什么的也不懂, 但是搜索到的信息告诉我要求使用django的appengine变种什么的, 反正就是看起来很烦, 烦到你最后放弃了. pyramid的cookbook写用什么monkey做部署, 我以为可以了, 结果monkey是远古时代的代码, 然后这就给我一种错觉, GAE上的python版本是2.4的, 然后我就没做具体了解了.

然后就到了上次北京GDG devfest了, 因为我有看到appengine的相关信息, 我还很期待会上可以交流了解下. 结果人家是Java党= =

回去之后就开始翻developer上面的文档了. 这个时候无意中搜索pyramid gae的时候看到cookbook上面有个buildout的部署方式, 很新奇, 但是尝试失败, 提示找不到setuptools, 搜遍仍然找不到解决之道.

废话不多说, 马上干货

Buildout

buildout是什么呢? 我把它看作是java下面的maven, 是不同于pip或者easy_install的包管理方式. 我相信很多人都是virtualenv党, 或者说像我这样再加上virtualenvwrapper的. 但是大家会发现什么呢? 我觉得很乱.

什么.egg-link .egg-info .py .pyc .pth还有各种文件夹, 我压根不知道我装了什么.
再者, 你pip install -U xxx的时候, 连开发版本都安装进去了. 就算你写了install_requires, python setup.py develop的时候同样会把依赖的开发版本, 但是你知道这肯定不是我要的. 有人又要说了, 我们可以输出requirement.txt让别人准确安装, 但是这里又要说了, 你可能安装了多余的包, 另外, 升级还是没办法解决.

buildout就是写一些配置文件, 控制你要安装的包, 但是你又可以写一些recipe, 我姑且称之为处方, 做任何你想做的东西.

用buildout我们就是要抛弃pip和easy_install这样的安装方式, 所以准备好你的环境.
